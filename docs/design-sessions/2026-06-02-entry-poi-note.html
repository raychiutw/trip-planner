EditEntryPage — 每個正選/備選景點各自可編輯備註

場景：一個午餐停留點，正選「暖暮拉麵 牧志店」+ 2 個備選候選餐廳。每個 POI 各自一條備註（trip_entry_pois.note），底部的「整體備註」section 移除。

3 個 variant 差在「per-POI 備註」怎麼呈現/編輯。其餘（時間 / 移動方式 / 危險區）不變，未畫出。

Terracotta tokens（真實 token 值） 44px tap target 無框線風格 ≤980px 自動單欄堆疊
Variant A

常駐備註欄

每個 POI 下方常駐一個備註欄，永遠可直接打字。最直接，但 N+1 個欄位讓頁面變高、空欄多時視覺噪音大。

Variant B ★ 推薦

點擊編輯備註行

沿用 TimelineRail 既有 note 模式：備註以一行文字顯示（空 → 「+ 加備註」），點擊就地展開 textarea + autosave。緊湊、有備註才佔空間、跨頁一致。

Variant C

展開式備註鈕

每個 POI 一顆「備註」chip（有內容時帶圓點），點擊展開編輯區、預設收合。列表最緊湊，但備註內容預設看不到 — 不利「比較候選的註記」。
