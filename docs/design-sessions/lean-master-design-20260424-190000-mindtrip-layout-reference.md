### Pane 規格

| Pane | Width | Position | Role |
|------|-------|----------|------|
| **Left sidebar** | **~240px fixed**（不 fluid） | `position: sticky; left: 0` | Global navigation + user |
| **Main content** | `calc(100vw - 240px - ${sheetWidth})` fluid | center column | Current task UI（chat / trip creation / empty state） |
| **Right trip sheet** | **~780px（40-45vw）**，可 collapse 至 0 | `position: sticky; right: 0` | Trip context / metadata / content tabs |

### Sidebar 內部 spec（Image 1, 7, 11）

```
Sidebar (240px, full height, dark bg)
├ Padding: 16px left/right, 24px top
├ Logo row: ✦ mindtrip. + collapse arrow (40px height)
├ Spacer: 32px
├ Nav items: 48px height each, 14px padding-left
│   ├ Icon (20px) + Label (14px font)
│   └ Active: subtle bg highlight, no border
├ "New chat" CTA button (full-width, pill-shape, dark outline)
├ Bottom: Traveler chip (40px avatar + name + ...更多)
└ Footer: Company · Contact · Help · Terms · Privacy (12px muted)
```

### Right Sheet 內部 spec（Image 11）

```
Right Sheet (~780px wide, full height)
├ Sheet top bar: [X close] [← shrink] [share] [...] [Sign up banner] [Go to trip]
│   └ Height ~56px
├ Trip title area (~80px)
│   ├ "Trip to Okinawa Prefecture" (28px bold)
├ Metadata chip row (~48px)
│   ├ Okinawa Prefecture | When | 1 traveler | Budget | Preferences
│   └ Selected: bordered pill; Unselected: borderless text
├ Content tab row (~48px, underline style)
│   ├ Itinerary | Ideas | Bookings | Calendar | Chats | Media
│   └ Active: white text + underline; Inactive: muted
├ Tab content area (scrollable)
│   ├ Varies per tab
│   └ Padding: 24px
```

### 桌機關鍵 behaviors

1. **Right sheet 可 collapse**：close X → sheet 隱藏 + main 擴寬；shrink 箭頭 → sheet 收窄但保留輪廓（collapse to min width）
2. **URL 反映 state**：`?tripSheet=1&tripTab=ideas` → 可直接 share link 到特定 tab（Image 7 URL 可見）
3. **Sidebar 永遠 sticky**，即使 modal 開啟也可見（僅被 modal overlay 遮住）
4. **Modal overlay**：背景 dim（~60% opacity black）+ modal centered，底下 sidebar + main 仍視覺可見

```

├ [X close] top-left (24px padding)
```

- Bottom sheet 留給 secondary context（view details、quick filter）

### 7. Ideas Empty State + Add Zone（Image 12）

Ideas section 空時顯示**一個大 Add card**（不是一個文字連結）：
```
┌──────────────────────────┐

│          +               │  ← icon 64px，居中
│         Add              │  ← label 14px，icon 下方
│                          │

    寬 ~180px × 高 ~180px
    Border: dashed or solid subtle

    Drop zone = drag Ideas card into here
```

**Ideas 有內容時**：Add card 消失或縮小變 chip，上方堆疊 idea cards。

### 8. Drag-to-Promote / Drag-to-Reorder Pattern（使用者觀察）

Mindtrip 支援 **4 種 drag 動作**，trip-planner 實作 Ideas + Itinerary 時要全覆蓋：

| # | 動作 | Drag Source | Drop Target | 結果 |
|---|------|-------------|-------------|------|
| 1 | **Promote**（核心） | Ideas card | Day N 的時段 | Ideas 項變 Itinerary entry，`trip_ideas.promoted_to_entry_id` 綁 FK |
| 2 | **Reorder within Day** | Itinerary entry | 同 Day 其他 entry 位置 | 改 `start_time` + `order_in_day` |
| 3 | **Cross-Day move** | Itinerary entry | 另一個 Day | 改 `day_id` + 新 `start_time`（default 同時段或 smart placement）|
| 4 | **Demote**（選配） | Itinerary entry | Ideas section | Entry → Ideas，清 day/time，保留 poi 關聯 |

**建議實作**：
- **Library**：[`@dnd-kit/core`](https://dndkit.com/)（React 生態標準，已 prod-ready，支援 mouse + touch + keyboard accessibility）
  - `@dnd-kit/sortable` for within-Day reorder
  - `@dnd-kit/modifiers` for Y-axis-only drag（避免誤拖出 Day group）
- **觸覺回饋**：drag hover drop zone 時，drop zone 加亮（border highlight + subtle bg tint）
- **時段智慧放置**：drop 到 Day N 時若沒指定時段 → default 排到該 Day 最後 entry 之後 1 小時，或早上首時段
- **Mobile touch**：long-press 100-200ms 啟 drag（避免誤觸），drag ghost 跟手指
- **Undo**：drag 完成後 toast「已移動 XX 到 Day 2，undo」5 秒可 revert
- **Conflict**：drop 時若時段衝突（與既有 entry 時間重疊）→ 彈 modal「這時段有 X，要換位置還是併排？」

**trip-planner 現狀 gap**：
- 目前 `trip_entries` 有 `start_time / day_id` 但 **沒 `order_in_day`** — 要加 column
- 無 drag library 套件 — 要 install `@dnd-kit/core + sortable`
- 無 Ideas table — 要建（見 Mindtrip benchmark doc Approach B）

## Responsive Breakpoint 策略

```
Desktop  (≥ 1024px) : 3-pane full layout (sidebar + main + sheet)
Tablet   (768-1023) : 2-pane（sidebar + main），sheet 變 full-screen overlay or bottom sheet
Mobile   (< 768px)  : 1-pane + bottom nav + full-screen modal
```

**建議 trip-planner breakpoint**（align Mindtrip）：
```css
/* trip-planner 現有 DESIGN.md: ≥1024px 2-col */
@media (min-width: 1280px) { /* 3-pane full */ }
@media (min-width: 1024px) and (max-width: 1279px) { /* 2-pane: sidebar + main, sheet become floating panel */ }
@media (max-width: 1023px) { /* 1-pane + bottom nav */ }
```

## URL-Driven State Convention

從 Image 11 URL `/chat/6015558?tripSheet=1&tripTab=itinerary` 推測 Mindtrip 的 URL pattern：

| URL Component | 意義 |
|---------------|------|
| `/chat/{id}` | chat/trip entity id |
| `?tripSheet=1` | 右側 trip sheet 開啟 (0 or absent = closed) |
| `?tripTab=itinerary` | trip sheet 當前 tab |
| `?tripFilter=...` | (inferred) filter state |

**Benefit**：
- Deep-linkable：share 特定 state 的 URL
- Browser back/forward 正常工作
- Reload 不 lost state

**對 trip-planner 建議**：
```
/trip/:id                              → full trip timeline
/trip/:id?sheet=map                    → 右 map panel open
/trip/:id?sheet=ideas&tab=suggested    → ideas tab with filter
/trip/:id/new-day                      → sub-route for modal (not query)
```

Convention：**query param for reversible state / open-close UI**; **sub-route for full destination changes**.

## Adaptation Playbook — trip-planner 套用 Mindtrip 的具體方式

### A1. 桌機布局改 3-pane（長期）

**當前 trip-planner 是左 timeline + 右 map rail 的 2-col**。想擴功能（chat / ideas / bookings）要多個內容區。改成：

```
Sidebar (240px)       Main (fluid)             Right Sheet (~780px)
┌──────┐              ┌────────────────┐       ┌───────────────┐
│ tp   │              │ Timeline chat   │       │ Tabs:          │
│      │              │ or AI edit      │       │  Map           │
│ 行程  │              │                 │       │  Ideas         │
│ 探索  │              │                 │       │  Bookings      │
│ 收藏  │              │                 │       │  ...           │
│ ...  │              │                 │       │                │
└──────┘              └────────────────┘       └───────────────┘
```

**Sidebar 7 個 nav 對照 trip-planner 需求**：
```
行程 Trips       → /manage
探索 Explore     → /explore（新）
收藏 Saved       → /saved（新，Mindtrip 式 collection）
想法 Ideas       → sheet tab（per-trip）
通知 Updates     → 旅伴請求 / 更新
靈感 Inspiration → public trips（未來）
建立 Create      → launch trip modal
```

### A2. 右 sheet with URL-driven tabs

現 `/trip/:id/map` 變 `/trip/:id?sheet=map`。新增 `sheet=ideas` / `sheet=chat`。
好處：深連結 / back button 正常 / 可 bookmark。

### A3. 手機 Explore（新）

```
/explore 路由（mobile-only 初期）
├ 上：地圖 + 搜尋 overlay + 拖把 handle
├ 下：location chip + category tabs + POI card carousel
│   └ 卡片有 ❤ + ⊕ icon（save / add-to-current-trip）
```

這個需要 **POI global index** — 即使 trip-planner 現在 POI 都綁 trip，要做 Explore 就得有 cross-trip POI browse。

### A4. 手機 modal type

trip-planner 現手機 modal 用什麼方式沒定。**抄 Mindtrip：full-screen cover**（not bottom sheet）for：
- 新增 trip
- 新增 day / entry（CRUD 表單）
- 登入

Bottom sheet 留給 **secondary context**（POI detail peek / filter quick action / share picker）。

### A5. Modal 規格統一

| Trip-planner modal | Desktop width | Mobile | 範例 |
|--------------------|---------------|--------|------|
| Create trip | ~1000px centered (可有左側插畫) | Full-screen cover | Mindtrip "Where to?" 同規 |
| Date picker | ~900px | Full-screen cover | Mindtrip "When" 同規 |
| Confirm delete | ~480px centered | ~80% width centered | Destructive confirm |
| POI detail | ~640px centered | Bottom sheet ~80vh | Peek content，不是行動 modal |

## 差異化 — trip-planner 不該完全複製的地方

| Mindtrip 做法 | trip-planner 建議 | 為何不同 |
|---------------|------------------|---------|
| Pure black dark theme | **Terracotta warm dark**（`#1C140F`） | 旅遊雜誌暖調 vs tech utility 冷調 |
| 7 sidebar nav | **簡化 5 個**（砍 Updates/Inspiration 到後期） | 早期 focus |
| 6 mobile tab（含 Log in）| **保 4-tab**，登入不佔 tab 位 | tab 多會擠 + 降 tap target |
| Infinite horizontal chips 「For you / Things to do / ...」 | **限 3-4 main category** | trip-planner POI 類型較窄（景點 / 餐廳 / 飯店 / 購物） |
| Modal 有大插畫 side（`1100px` 寬） | **Modal 窄一半，右配圖**（`720-800px`） | 減空氣，保暖調 |
| 非品牌 emoji 多（👋 / 🎈 / 🛍️） | **保持 emoji 節制**（DESIGN.md 有 emoji policy） | Anti-slop 紀律 |
| `?tripTab=ideas` URL param 驅動 tab | 沿用 ✅ | 純 pattern，無異議 |

## Component Checklist — trip-planner 可直接引用做 UI

實作 trip-planner 新 page / feature 時，逐項對照 Mindtrip pattern：

### ☑ 每 new page 必有
- [ ] Desktop: sidebar (240px) + main + (optional) right sheet
- [ ] Mobile: content + bottom nav
- [ ] Heading + 副標 + primary CTA（如是 empty state）
- [ ] 回應 2 種 viewport：`< 768` mobile / `≥ 1024` desktop（1024-1279 trans zone）

### ☑ 每 modal 必有
- [ ] Desktop: centered, max-width 1000px, close X top-left
- [ ] Mobile: full-screen cover (不要 bottom sheet 除非 secondary context)
- [ ] Sticky bottom CTA button
- [ ] 按 ESC / tap backdrop 關閉

### ☑ 每 POI card 必有
- [ ] 圖片 cover
- [ ] Heart icon top-left（save to global）
- [ ] Plus icon top-right（add to current trip）
- [ ] 下方標題 14-16px（2-line truncate）
- [ ] Tap card → detail bottom sheet（mobile）or detail modal（desktop）

### ☑ 每 trip sheet 必有
- [ ] 可 collapse（close X + shrink 箭頭）
- [ ] URL 反映 open/close + current tab
- [ ] Top 為 metadata chips，Bottom 為 content tabs

## Appendix: Overlay Pattern Rules（待 merge 進 DESIGN.md）

Overlay = 任何遮蓋主內容的 UI layer（modal / bottom sheet / full-screen cover）。用錯 pattern 會破壞使用者心智模型。以下為 canonical rules：

### 決策樹

**1. 使用者要做 primary action（完整注意力 + 表單輸入）？**
- 桌機 → **Centered Modal**（max-width 1000px, rounded 12-16px, backdrop dim 60%）
- 手機 → **Full-screen Cover**（100vw × 100vh, close X top-left）
- 例子：建立 trip / 登入 / 選日期 / 編輯 entry 時間

**2. 使用者要快速看內容但不做 primary action（peek）？**
- 桌機 → **Popover**（小 floating panel，anchored to trigger）
- 手機 → **Bottom Sheet**（drag-able, ~60-80vh max）
- 例子：POI detail peek / 快速 filter preview / 分享選單

**3. 使用者要 destructive confirm（刪除 / 不可逆動作）？**
- 桌機 → **Small Centered Modal**（max-width 480px）
- 手機 → **Small Centered Modal** or Bottom Sheet
- 例子：確認刪除 trip / 登出 / 清除草稿

**4. 多步流程 wizard？**
- 桌機 → Centered Modal with progress indicator（步驟點 x/y）
- 手機 → Full-screen Cover with back button（header 顯示「Step 2 of 3」）
- 例子：Onboarding 三步 / 新 trip 三步（Destination → Timing → Preferences）

### Pattern 對照表

| Scenario | Desktop | Mobile | Max Width | Close Method |
|----------|---------|--------|-----------|--------------|
| Primary form | Centered modal | Full-screen cover | 1000px | X top-left + ESC + backdrop tap |
| Peek content | Popover | Bottom sheet | 600px / 80vh | ESC / swipe down / tap outside |
| Confirm destructive | Small modal | Small centered modal | 480px | X + Cancel button |
| Multi-step wizard | Centered modal | Full-screen cover | 1000px | X + back step button |
| Filter / share | Small modal | Bottom sheet | 400px / 60vh | Apply button or tap outside |

### 禁止（anti-patterns）

- ❌ **桌機用 Bottom Sheet** — bottom sheet 是 mobile-only pattern；桌機用 popover 或 modal
- ❌ **同一個流程混 Modal 和 Bottom Sheet** — 始末 consistent，避免使用者 mental model 斷裂
- ❌ **無 close method 的 overlay** — 永遠要至少一種（X button / ESC / backdrop tap / swipe down）
- ❌ **Modal 裡面再開 Modal**（stacked）— 重新設計原 flow，通常表示 scope 太大
- ❌ **Full-screen cover 在桌機用** — 桌機至少讓 sidebar 可見，除非是 immersive content（地圖 full-screen view）

### 搭其他 locked decisions 的一致性

- **3-pane 桌機 layout**（Q1）：modal 開啟時 sidebar 不應被遮，modal centered 在 main + sheet 區域
- **Bottom nav 手機常駐**（Q3）：full-screen cover 開啟時 bottom nav 被 overlay 遮（自然）；bottom sheet 開啟時 nav 上方被 sheet 部分遮蓋 OK
- **Query param URL state**（Q4）：primary form modal 可用 URL driver 如 `?modal=create-trip`；peek overlay 不進 URL（短暫）
