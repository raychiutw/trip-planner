Tripline V3 rev2 — 三欄 + 底部 tab + 手機 C

2026-07-15 · sidebar 展開常駐 · 主功能→底部浮動玻璃 tab · 中欄操作→右欄 · 手機 = C·Map First Drawer

帳號位置（owner 2026-07-15 更正）： 桌機 = 左下角 sidebar account chip（維持現行 DESIGN.md:358，不動）； 手機 = titlebar 右上角。 兩邊底部 tab 都只放 4 個主功能（聊天／行程／地圖／收藏），帳號不佔 tab slot。
tab 不滿版：浮動膠囊置中（iOS 26 Apple Music 形制）。不滿版不只是造型 —— 左右露出地圖，玻璃才有東西可透、才看得出內容從底下流過。滿版貼底的玻璃效果為零。
板子的 teal 只取結構不取色：C 來自 .flutter 專案用青綠系；此處套 V3 中性深色 + 柔褐 accent，避免兩專案品牌分裂。

玻璃的實話：玻璃要有東西可透才成立。左欄 sidebar 背後什麼都沒有（它貼在視窗最左緣）， 上玻璃只會得到「有色板」—— 就是你現在 --color-glass-nav 92% 那個問題的翻版。 這裡仍照你要求上了玻璃，但它的價值幾乎只有 rim 高光。
玻璃真正賺到的地方有三個，都在此 mockup 內： ① 浮在地圖上的「行程操作」面板（Apple 官方的 clear variant over media 情境）、 ② 底部浮動 tab（內容從底下流過）、 ③ 手機的 Map First Drawer（地圖在底下透上來）。

Section 01

桌機三欄 — 右欄是堆疊面板（地圖 → 滿版操作 → 再蓋一層 → 不封頂）

中欄是「旅遊路線」不是播放清單（owner）：連續軌線串起停留點、移動段用虛線 —— 刻意呼應右欄地圖的 dashed polyline，讓兩欄是同一條路線的兩種視圖。 日首尾是飯店（沿用 DESIGN.md:190「hotel 是當日 polyline 的起點」既有語意）。 原有 action 保留：拖拉排序（hover 出 grip）／展開收合／複製到其他天／移到其他天／編輯備註／刪除景點（放大檢視移除 —— POI 沒有照片資料，該鈕只會開出 placeholder）， 外加時間 chip 就地改起訖（v2.55.51）、備選 + 設為正選、交通 pill 可點換方式。

右欄堆疊語意（owner 2026-07-15）：點選景點 → 面板滿版蓋滿整欄（不是浮在角落的 inset 卡）， 右上【關閉】；再蓋一層 → 左上【回前頁】+ 右上【關閉】。這是 iOS navigation stack：第 2 層 modal（由下往上）、 第 3 層以後 push（由右往左），下層退後 + 壓暗做深度暗示。

不封頂在三層（owner 2026-07-15：「有 4 層就堆疊」）—— 規則不是「L3 特例、L4 再特例」，而是照階差： depth == stack 在位、> stack 收在右邊待命、< stack 往後退， 退的量與暗度跟差幾階成正比（差 1 階 -14% / .72、差 2 階 -25% / .48）。加第五層只要照這個級數往下寫。 L4 的動線是被需求逼出來的、不是為了展示堆疊：L2 停留點操作 → L3 換景點 → L4 該景點詳情， 換景點不該盲換，按下去前要能看一眼。「‹」永遠只退一階（L4→L3），「✕」才收掉整疊回地圖。 按下方四顆鈕走一遍 —— 退後與壓暗只在轉場動畫過程中看得到（上層滿版覆蓋，靜止時看不見，這點與 iOS push 一致）。
