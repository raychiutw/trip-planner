Section 5 · 實作

提案的 token 結構

現況 --blur-glass 只統一了六層裡最不重要的一層，其餘五層各自漂移 —— 這是 drift 的結構性原因。改成兩組完整材質，每組是一整包。

Token值用在哪
材質 A · GLASS（media-rich 底層限定）
--material-glass-bgsecondary 58%地圖頁 TitleBar、地圖頁底部膠囊、地圖浮動控制項
--material-glass-blurblur(20px) saturate(180%)
--material-glass-rim1px solid rgba(255,255,255,.42)
--material-glass-specularinset 0 1px 0 rgba(255,255,255,.55)
--material-glass-shadow0 12px 32px …26 · 0 2px 8px …14
材質 B · SOLID（flat cream 底層）
--material-solid-bgvar(--color-secondary) 不透明聊天／行程列表／表單／收藏／設定的 TitleBar 與底部膠囊、page bottom bar
--material-solid-rimV3 = transparent（無框）
--material-solid-specularinset 0 1px 0 rgba(255,255,255,.85)
--material-solid-shadow雙層 0 8px 18px + 0 22px 44px
形制
--chrome-inset21px膠囊左／右／下 inset（HIG iOS 26 量測值）
--font-size-caption211px（已存在，未被用）tab label — 取代誤用的 --font-size-eyebrow（10px）
退役
--blur-glass: 14px單一 blur 值被兩組完整材質取代
--color-glass-navbackground 92%無 consumer 存留 → 一併清掉
順帶收掉的 drift：現況五種不透明度（transparent / 80% / 88% / 94% / 97%） 加上桌機 sidebar 的第六種（72% + blur(30px)）全部收斂進上表兩組。 Sidebar 的 macOS vibrancy 屬 DESIGN.md L283 已核可的刻意例外，維持不動。
Section 6 · 無障礙

三個 fallback（目前全 codebase 為 0）

prefers-reduced-transparency 與 prefers-contrast 在整個 repo 零命中，只有 prefers-reduced-motion 有實作。Apple 這三項是系統自動套用，Web 必須手寫。用上方控制列切換驗證。

系統設定Apple 行為本專案對應
reduced-transparency 玻璃變得更霜化，遮蔽更多背後內容 GLASS → secondary 96% + blur(6px)；SOLID 本就不透明，不變
contrast: more 元素變成主要為黑或白，加對比色邊框 兩組材質 → 全不透明 background + 2px solid foreground 邊框，specular 關閉
reduced-motion 停用彈性屬性，降低效果強度 scroll edge 的 200ms 材質淡入 → 直接切換（已有 prefers-reduced-motion 基礎設施）
對比度注意：玻璃版的 tab label 是 --color-muted 疊在半透明底上， 底層是地圖圖磚 —— 亮綠地／深水域都可能出現。實作時需對最壞情況底色驗 WCAG AA 4.5:1， 不能只對這個 mockup 的假地圖驗。這是 /tp-code-verify 的 CSS HIG 檢查項。
