是否要強制一致？兩個選項

選項 1 — 強制一致（全部用 Style A）

把 Style B / C / D 全部砍掉，所有 input 統一 Style A。變化：

  • EditEntryPage 「抵達 10:45 AM」失去 22px bold 視覺重點，變成跟其他 input 一樣大小 — 時間欄位視覺權重下降
  • Favorites form 變大 (10→12 padding, 1→1.5 border, radius 12→16)
  • Shift modal date 不再 center align — 跟 form input 一樣 left align

適合：強調「全 app 視覺統一」/ 不太在意 visual hierarchy。

選項 2 — 保持現狀（4 種 styles，intentional variation）

各 page-scoped class 保留差異，因為都是有意義的視覺層級：

  • Style A 是 default 通用 form (NewTripPage / LoginPage / 一般 text)
  • Style B (Favorites) 較 compact — 適合 modal-like form
  • Style C (Time card) 強調時間數字 — EditEntryPage UX 重點
  • Style D (Shift modal) 居中 短 date 字串 — modal 視覺對稱

適合：保留 page-specific UX 強調。

建議：選項 2（現狀）— 因為 Style C 是 EditEntryPage 時間 UX 重點（22px bold 強調），改回 16px 會失去視覺權重。其他 (A/B/D) 差異本來就很小（padding 2px 等），用戶不易察覺。
