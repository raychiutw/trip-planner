強制 Style C 全域 — 視覺預覽

把全部 input 套 Style C (no border / transparent / 22px bold)。下方各種真實 use case 看效果。

⚠️ 已知 UX 問題：22px bold 對長文字（email / 描述 / 標題）破板嚴重。對短值（time / date / number）有效果。

✓ 短值（time / date / number）— OK，視覺強烈

這是 Style C 原本設計目的。

~ 中等長度（trip 名稱 / 描述短）— 還可

字元數 < 20 字 OK，超過會擠或撞框。

✗ 長值（email / 完整 trip 名 / 地址）— 破板

22px 把字撐太大，超出 input 寬度時 user 看不到尾巴。

✗✗ Textarea（備註 / 描述）— 災難

22px bold 段落文字基本上沒辦法讀。SEO description 失去意義。

✗ 搜尋（POI / 收藏 / trips）— 不適合

22px bold + label 把搜尋框做大，跟既有 inline 搜尋 UX (chip-style) 衝突。

~ Select dropdown — 看起來 OK

但 select 跟 input 同樣大會增加垂直 form 長度。

實際應用：LoginPage 強制 Style C

原本 email / password 兩個 flat input → 變成兩張 card。表單變高，但結構更清楚。
