4. 風險與緩解

R1 — Effect 執行順序變動
React 對同 component 內 hook 的 effect 依宣告順序執行。重組成 3 個 custom hook 時，內部 effect 順序若無意換位，可能讓 markersRef setup 在 viewport fit 之後跑 → fit 時 marker 還沒 mount → bounds 算錯。
M1：useMapMarkers 第一個 call、useMapViewport 第二個、useMapSegments 第三個 — 對齊原檔 line 437 → 514 → 557 順序。內部 effect 也按原檔順序。
R2 — useEffect deps 漏掉或多加
把 effect body 搬進 custom hook 時，dep array 必須完全比照原本（含 eslint-disable 的部分）。否則 re-run frequency 變化 → 多餘 marker 重建或漏更新。
M2：逐 effect copy-paste，dep array 一字不動。// eslint-disable 註解也搬。Refactor 後 grep diff 確認 dep arrays 完全相同。
R3 — StrictMode double-mount 行為改變
Dev mode 下 StrictMode 跑 effect 兩次。若 hook 內部 ref 管理跟 cleanup 順序變化， 可能多殘留一個 marker 或 polyline。
M3：手動 dev mode 測 detail / overview 切換、focus 變化、pan/zoom； 檢查 markersRef 與 google.maps 殘留 instance count；新 e2e smoke test 覆蓋。
