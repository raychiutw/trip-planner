行為規格（對照 Flutter trip_map_screen.dart / map_canvas_mobile.dart / google_poi_accessory_card.dart）

  • 攔截點：Google Maps JS API map.addListener('click', e => e.placeId)（對等 Flutter google_maps_flutter 的 onPoiClicked）。web 目前 useGoogleMap.ts 設 clickableIcons:false 整個關掉，本次改為可選開啟（僅地圖頁面傳入 onPoiClick 時才開，行程建立時的地點挑選地圖 LocationPickerMap 不受影響）。
  • 資料落差（web 特有，Flutter 沒有這步）：JS API 的 IconMouseEvent 只給 placeId＋座標，沒有店名（原生 SDK 端就地取得，免費）。web 要顯示店名需多打一次既有 GET /api/places/resolve?placeId=（既有端點，Place Details，$17/1000 次、user 500 次/24h rate limit）。點圖示的當下先顯示「載入地點名稱…」，resolve 回來再補上。
  • 關閉：點地圖空白處（無 placeId 的 click）、或按卡片 X → 換回原本的行程 POI 卡橫向捲動。
  • 「在 Google 地圖開啟」：組 https://www.google.com/maps/search/?api=1&query=<name或lat,lng>&query_place_id=<placeId>，target="_blank" 開新分頁（Flutter 端是 url_launcher 開外部 app／瀏覽器，離開 app；web 對應行為是開新分頁，不導離 Tripline 分頁）。重用既有 buildMapsUrl()（src/lib/mapsUrl.ts），加一個可選 placeId 參數。
  • 不含：加入行程、星等、地址、營業時間 — Flutter 卡片本身就只有「名稱＋關閉＋開地圖」三件事，這次對齊不加碼。
  • 觸控目標：關閉鈕與開地圖按鈕 ≥44px（對齊 HIG，Flutter 用 TpSpacing.tapMin）。
