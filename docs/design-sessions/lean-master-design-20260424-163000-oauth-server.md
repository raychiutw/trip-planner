## Problem Statement

trip-planner 從 Cloudflare Access email 白名單（2-user 私人工具）轉型為 **完整 identity system + OAuth 2.0 Authorization Server**：

1. 任何人可 Google OpenID 登入
2. 任何人可自建帳號密碼註冊
3. 忘記密碼可 email reset
4. 第三方 app 可接「Sign in with Trip-Planner」

這個 scope = **trip-planner 同時是 SaaS 產品 + identity provider + data provider**。

## Recommended Architecture: **@openauthjs/openauth + 客製 UI**

### 為什麼

1. **Cloudflare Workers 原生** — fit 你現有 stack，不換 infra
2. **OAuth 2.0 Authorization Server built-in** — authorize / token / userinfo / revoke / JWKS endpoints 都有
3. **Multi-provider pluggable** — Google OIDC adapter + Password adapter + 未來 Apple / LINE 都是 `providers: [GoogleProvider({...}), PasswordProvider({...})]` 這種 API
4. **PKCE 強制** — security best practice 內建
5. **Token 簽章 + JWKS 發布** — 不用自己處理 key rotation 複雜度
6. **D1-compatible storage adapter** — session / consent / auth code 存你的 D1
7. **開源 MIT** — 可 fork，無 lock-in
8. **sst.dev 團隊維護** — 活躍專案，2024 起

### 分工

| 工作 | @openauthjs/openauth | 你自寫 |
|------|---------------------|--------|
| OAuth 2.0 protocol compliance | ✅ | — |
| Authorization Code Flow + PKCE | ✅ | — |
| Token signing (RS256) + JWKS publish | ✅ | — |
| Google OIDC adapter | ✅ | — |
| Password provider (bcrypt/Argon2) | ✅ | — |
| Session management | ✅ | — |
| D1 storage | ✅（adapter） | 寫 migration |
| Consent screen UI | — | ✅ Terracotta 風格 |
| Forgot password UI + email 整合 | — | ✅ |
| Signup / login UI | — | ✅ |
| `/admin` client management UI | — | ✅ |
| Multi-device session UI (`/settings/sessions`) | — | ✅ |
| Rate limiting（敏感 endpoints） | — | ✅ Cloudflare WAF + 自己邏輯 |
| Email delivery（verify / forgot） | — | ✅ Cloudflare Email Routing 或 Resend |

**工作量收縮**：原 V2 自建 = 8-12 週。用 @openauthjs/openauth = **5-7 週**（省 30-40%）。

### 為什麼不用 Auth0 / Clerk

你先前 V1 選 B 時說「之後可以接其他 opid 或者自建帳號密碼」→ 不想 lock-in。Auth0 / Clerk 把身分資料放他們 server，未來搬家 = 所有 client apps 重新 integrate（因為 issuer 換了）。@openauthjs/openauth 你自己 host，issuer URL 就是你的 domain，永遠可控。

## Schema (與 @openauthjs/openauth 搭配)

### Identity Core

```sql
-- Migration 0028_identity_core.sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  primary_email TEXT NOT NULL UNIQUE,
  email_verified INTEGER NOT NULL DEFAULT 0,
  name TEXT,
  avatar_url TEXT,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  disabled_at TEXT
);

CREATE TABLE auth_identities (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  provider TEXT NOT NULL CHECK (provider IN ('google', 'local', 'apple', 'line')),
  provider_sub TEXT NOT NULL,              -- Google sub / email for local / Apple sub
  email_snapshot TEXT,                      -- email at link time (non-canonical)
  -- Local password provider only
  password_hash TEXT,                       -- Argon2id hash; NULL for other providers
  password_updated_at TEXT,                 -- track for rotation policy
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  UNIQUE (provider, provider_sub)
);

CREATE INDEX idx_auth_identities_user_id ON auth_identities(user_id);

CREATE TABLE sessions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  token_hash TEXT NOT NULL UNIQUE,          -- SHA-256(opaque_token)
  expires_at TEXT NOT NULL,
  last_seen_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  revoked_at TEXT,
  user_agent TEXT,
  ip_hash TEXT,                             -- SHA-256(ip) for audit
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_sessions_token_hash ON sessions(token_hash);
CREATE INDEX idx_sessions_user_id ON sessions(user_id);
```

### Password & Email Verification

```sql

CREATE TABLE email_verifications (
  token_hash TEXT PRIMARY KEY,              -- SHA-256(token)
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  email TEXT NOT NULL,                      -- the email being verified (could be new)

  token_hash TEXT PRIMARY KEY,              -- SHA-256(reset_token)
  user_id INTEGER NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  email_at_request TEXT NOT NULL,           -- email at time of request (audit)
  expires_at TEXT NOT NULL,                 -- 15-30 min
  used_at TEXT,                             -- 一次性
  ip_hash TEXT NOT NULL,                    -- 申請時的 IP hash (anti-abuse)
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_password_resets_user ON password_resets(user_id, used_at);

CREATE TABLE auth_rate_limits (
  key TEXT PRIMARY KEY,                     -- e.g. "signup:ip:1.2.3.4" / "login:email:foo@bar.com"
  attempts INTEGER NOT NULL DEFAULT 1,
  window_start TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  blocked_until TEXT
);
```

### OAuth Authorization Server

```sql

CREATE TABLE oauth_clients (
  client_id TEXT PRIMARY KEY,
  client_secret_hash TEXT NOT NULL,         -- Argon2id
  name TEXT NOT NULL,
  redirect_uris TEXT NOT NULL,              -- JSON array; exact match enforced
  allowed_scopes TEXT NOT NULL,             -- JSON array

  kid TEXT PRIMARY KEY,
  algorithm TEXT NOT NULL DEFAULT 'RS256',
  public_key TEXT NOT NULL,                 -- PEM
  private_key_encrypted TEXT NOT NULL,      -- encrypted with env secret (AES-GCM)
  active INTEGER NOT NULL DEFAULT 1,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  rotated_at TEXT
);

-- @openauthjs/openauth 管 auth_codes + access_tokens + consents，
-- 但 schema 需由你 provision（lib 提供 migration generator）
CREATE TABLE oauth_authorization_codes (
  code_hash TEXT PRIMARY KEY,
  client_id TEXT NOT NULL REFERENCES oauth_clients(client_id),
  user_id INTEGER NOT NULL REFERENCES users(id),
  redirect_uri TEXT NOT NULL,
  scopes TEXT NOT NULL,
  code_challenge TEXT NOT NULL,
  code_challenge_method TEXT NOT NULL DEFAULT 'S256',
  expires_at TEXT NOT NULL,                 -- 10 min
  used_at TEXT,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE oauth_access_tokens (
  jti TEXT PRIMARY KEY,
  client_id TEXT NOT NULL REFERENCES oauth_clients(client_id),
  user_id INTEGER NOT NULL REFERENCES users(id),
  scopes TEXT NOT NULL,
  expires_at TEXT NOT NULL,                 -- 1h
  revoked_at TEXT,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE oauth_consents (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL REFERENCES users(id),
  client_id TEXT NOT NULL REFERENCES oauth_clients(client_id),
  scopes TEXT NOT NULL,                     -- granted scopes (JSON)
  granted_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  revoked_at TEXT,
  UNIQUE (user_id, client_id)
);
```

## Provider Implementations

### 1. Google OpenID Connect

```typescript
// Using @openauthjs/openauth
import { GoogleProvider } from "@openauthjs/openauth/provider/google";

GoogleProvider({
  clientID: env.GOOGLE_CLIENT_ID,
  clientSecret: env.GOOGLE_CLIENT_SECRET,
  scopes: ["openid", "email", "profile"],
})
```

Flow：
1. `/auth/google` → redirect to accounts.google.com
2. Google callback → lib verify id_token（JWKS cache 自動）
3. Lib call your `onSuccess` handler → upsert `auth_identities(provider='google', provider_sub=sub)` + link to `users` by email
4. Lib issue opaque session cookie

### 2. Local Password

```typescript
import { PasswordProvider } from "@openauthjs/openauth/provider/password";

PasswordProvider({
  // lib handles bcrypt/Argon2 internally; you provide storage + email verify
  async hash(password) { /* Argon2id */ },
  async verify(hash, password) { /* Argon2id compare */ },
  async signup({ email, password }) { /* create user + send email verify */ },
  async login({ email, password }) { /* validate + issue session */ },
})
```

流程：
- Signup → 建 `users` row (`email_verified=0`) + `auth_identities(provider='local', provider_sub=email, password_hash=argon2)` + send `email_verifications` email
- 使用者點 email 連結 → `/auth/verify?token=X` → `email_verifications.used_at = NOW` + `users.email_verified = 1`
- Login → fetch `auth_identities` by email → Argon2 compare → issue session

**Argon2id 參數建議**：
- Memory: 64 MB
- Iterations: 3
- Parallelism: 4
- Output: 32 bytes
- （符合 OWASP 2024 recommendation）

### 3. Forgot Password Flow

```
POST /auth/password/forgot { email }
  ↓
若 email 存在:
  - 產 random 32-byte token
  - INSERT password_resets (token_hash=SHA256(token), expires_at=NOW+15min, ip_hash)
  - Send email with link: https://{domain}/auth/password/reset?token={token}
若 email 不存在:
  - **仍 return 200**（防止 email enumeration attack）
  - 不 insert 不 send

POST /auth/password/reset { token, new_password }
  ↓
  - Lookup password_resets by SHA256(token)
  - Check expires_at > NOW, used_at IS NULL, user not disabled
  - 驗 new_password strength（>=10 chars, not in HIBP top 100k）
  - UPDATE auth_identities SET password_hash = argon2(new) WHERE user_id = ... AND provider='local'
  - UPDATE password_resets SET used_at = NOW
  - UPDATE sessions SET revoked_at = NOW WHERE user_id = ...  -- 安全：所有裝置登出
  - Send confirmation email（inform user password changed）
```

**安全設計**：
- Token 一次性使用
- 15-30 min 短 TTL
- Reset 成功後 revoke 所有 active sessions（防已入侵攻擊者保持 access）
- IP + rate limit：同 IP 5 min 內最多 3 次 forgot 請求
- Email 不存在時 return 200 + constant-time delay（防 user enumeration）
- **不透過 URL query string 傳 token 的 reset confirmation**（referrer leak）— reset UI 是 POST form

### 4. OAuth 2.0 Authorization Server

由 @openauthjs/openauth 提供核心 endpoints：
- `GET /.well-known/openid-configuration` — OIDC Discovery
- `GET /.well-known/jwks.json` — 你的簽章公鑰
- `GET /oauth/authorize` — 啟動 auth flow
- `POST /oauth/token` — exchange code for tokens
- `GET /oauth/userinfo` — return user profile (Bearer token)
- `POST /oauth/revoke` — revoke token (RFC 7009)

你自己補：
- `/app/oauth/consent` — Terracotta 風格 consent screen UI
- `/admin/oauth/clients` — client management UI
- Custom scopes: `trips:read` / `trips:write`（若第三方要資料 access）

## Security

### 攻擊面 vs V1

| 維度 | V1 (OAuth Client only) | 本 plan (OAuth Server + Password) |
|------|----------------------|-----------------------------------|
| Brute force | 無（Google 管） | ✅ 需 rate limit login |
| Credential stuffing | 無 | ✅ 需 rate limit + HIBP check |
| Email enumeration | 無 | ✅ forgot flow constant-time 回應 |
| Password reset 濫用 | 無 | ✅ IP rate limit + short TTL + one-time |
| Email spoofing (verify bypass) | 無 | ✅ DKIM/SPF 設對；token 強度 |
| OAuth Server compromise | 無 | ✅ signing key rotation + RCE 防禦 |
| XSS in consent screen | 無 | ✅ client name / logo URL escape |
| Open redirect via redirect_uri | 無 | ✅ exact match enforced |

### Hardening 清單

- [ ] Argon2id 密碼 hash（不用 bcrypt — OWASP 2024）
- [ ] Password policy: >=10 chars, check HIBP pwned-passwords API
- [ ] Rate limit: login 5/min/email, signup 3/hour/IP, forgot 3/hour/email, token 60/min/client
- [ ] Email verification mandatory（`users.email_verified = 0` 不能 login）
- [ ] Session revoked on password change
- [ ] `__Host-session` cookie（SameSite=Lax, Secure, HttpOnly, no Domain attr）
- [ ] CSRF protection on all state-changing endpoints
- [ ] OAuth: PKCE mandatory, strict redirect_uri exact match, state param echo
- [ ] OAuth signing key: RSA 2048+ or EdDSA, private key encrypted at rest, rotate 90 days
- [ ] OAuth consent screen: escape all client-provided content (name, logo URL)
- [ ] Audit log: signup, login, logout, password change, OAuth grant, token issue, token revoke
- [ ] Failed login alert（email user on 5 consecutive failures）
- [ ] Anomaly detection（new IP + new country → email alert）

### 外部 Security Audit

**Ship prod 前必做**：
- Independent security review（付費服務如 Trail of Bits / Cure53，或社群如 [hackerone.com](https://hackerone.com) 公開 bug bounty）
- OIDC conformance test：[openid.net/certification](https://openid.net/certification)
- Run `/cso` full mode on gstack

## Open Questions

1. **Email provider**：Resend vs Cloudflare Email Routing vs SES？建議 Resend（workers-friendly、免費層、developer-first docs）
2. **Domain + cookie**：`trip-planner-dby.pages.dev` 還是 custom？`__Host-` prefix 會限制
3. **Signup email 驗證是否阻擋 login**：嚴格推薦 YES（未驗信不能用）— 防 spam 濫用
4. **Password 長度最小值**：10 vs 12 vs 14？建議 10 + HIBP check（HIBP check 是 stronger signal than raw length）
5. **Forgot password email 從哪發**：`no-reply@trip-planner-dby.pages.dev`？需設 SPF/DKIM
6. **Password reset token TTL**：15 min（security）vs 30 min（UX）？建議 15
7. **OAuth client provisioning**：UI-only vs API 支援 Dynamic Client Registration（RFC 7591）？建議 Phase 7 後再加 DCR
8. **Scope 設計**：`openid profile email` + `trips:read` + `trips:write` 夠？還要 `trips:own`（只能讀自己）？
9. **@openauthjs/openauth 版本 pinning**：什麼 version？Beta 還是 stable？需 test compatibility
10. **Google Cloud OAuth app 類型**：Web application，需 verified 嗎？（production 用會要 verify）
11. **Key rotation 策略**：90 天 auto / 手動 / 事件驅動？建議 90 天 + 手動 emergency rotation
12. **第三方 audit cost**：Trail of Bits ~$30k / 週，Cure53 ~$20k / 週，HackerOne BB 5k+/bug — 預算？
13. **Consent screen 預設 remember**：勾 or 不勾？UX vs security tradeoff
14. **Admin panel role split**：superadmin / support / developer（看 client manage）分權限？Phase 6 再決定

1. **「刪除 V1」**：你明確 prune scope。V1 的 SaaS + Dark mode + CRUD 不在這份 plan，代表你 focus 在 **auth 是核心基礎、其他 UX 層後補**。這是對的優先順序 — 沒 auth 其他都無意義。但這也承擔風險：**沒 UX 層 ship，auth system 本身 0 user-visible value**，你可能 4-6 週後有一個 perfect auth system 但 trip-planner 不能用。**建議**：Phase 7 ship 前先快速做一個 minimal login page + redirect-to-existing-app，讓 auth system 至少有入口。

2. **「用開源第三方套件也可以」**：你 explicit open to OSS，這是成熟判斷。純自建 OAuth Server 是「技術自尊」陷阱，很多人被困在那 6 個月。@openauthjs/openauth 省 30-40% 時程是具體價值。

3. **「自建帳號密碼 + 忘記密碼」**：你把這個跟 OAuth Server 綁一起規劃 — 表示你理解 multi-provider 是同一個 identity system 的不同接入點，不是 OAuth Server vs 自建 password 二擇一。schema 設計 `auth_identities(provider)` enum 正是 capture 這個。

4. **你選 autoplan 而不是直接 /opsx:propose**：你要在實作前把這份 plan review 到骨子裡。這跟你「一次到位」的傾向一致 — 寧可花 autoplan 的時間前期 review，也不要實作中才發現 architecture 問題。**尊重。這正是 autoplan 存在的原因**。

5. **你在 CEO gate override 兩 model consensus、在 final gate 又 override Eng+DX factual findings**：這是連續 3 次 user sovereignty 決定。Autoplan 的紀錄在此，未來你若某週覺得「當初為什麼沒擋我」— doc 有記錄，我有 flag，兩個 model 都留話。不是復仇，是讓未來的你有 context。

### 🔴 CRITICAL — 開工前必驗 / 必改

- [ ] **Spike 1-2 天**：驗證 `@openauthjs/openauth` 實際可用狀態 — D1 adapter 是否存在？仍 beta？版本 pin 什麼？**若 adapter 不存在 → 選一：自寫 D1 adapter / 改用 KV storage / 換 Lucia-style hand-roll / 用 Clerk 外包**（Codex web search: 官方 docs 推薦 KV，`wrangler.toml` 只 bind D1）
- [ ] **OAuth 不能共用 `_utils.ts` JSON helper**（會 camelCase 轉換 `access_token` → `accessToken`，破 RFC 6749 wire format）→ 建 `functions/api/_oauth_responder.ts` 專發 snake_case
- [ ] **OAuth 不能共用 `_errors.ts`**（nested format 不符 RFC 6749 flat error）→ OAuth endpoints 用 `{error, error_description, error_uri}` schema
- [ ] **公共 URL 定 one issuer base** — freeze `/oauth/*` 為 public path；`/functions/api/oauth/*` 是 internal routing，不對外 documented
- [ ] **Test plan 補完整 failing-test diagram**（違反全域 CLAUDE.md TDD）— minimum 20 red tests 見下方清單
- [ ] **RFC 6749 error catalog**：13 個 standard errors + `error_uri` 指向 `/docs/oauth/errors#<code>` — 開工前先 stub 所有 errors

### 🟠 HIGH — Phase 1 前 spec 完整

**Schema**
- [ ] `trip_permissions` 加 `user_id INTEGER REFERENCES users(id)` + backfill + OR logic（email OR user_id）
- [ ] `users.primary_email` **非 UNIQUE** — 改建 `user_emails(user_id, email, verified_at, is_primary)` 獨立 table（避免 email change 流程地獄）
- [ ] `auth_identities` 加 `CHECK ((provider='local' AND password_hash IS NOT NULL) OR (provider<>'local' AND password_hash IS NULL))`
- [ ] 1 user 只能有 1 個 `provider='local'` row — partial unique index
- [ ] `oauth_client_redirect_uris(client_id, uri)` 獨立 child table（不要 JSON blob）
- [ ] `oauth_client_scopes(client_id, scope)` 獨立 child table
- [ ] 加 `oauth_refresh_tokens(token_hash, client_id, user_id, access_token_jti, rotated_from, ...)` — OIDC clients expect offline_access
- [ ] `users.pending_email` for email change staging

**Security**
- [ ] Session rotation on **every** auth state transition（login / password reset / MFA / privilege change）
- [ ] Auth code atomic use: `UPDATE ... SET used_at=NOW WHERE code_hash=? AND used_at IS NULL RETURNING *` — detect zero rows = replay → revoke entire token family
- [ ] Signing key rotation: support N active keys with `status / not_before / retire_after`；JWKS publish 舊+新 until max(access_token TTL)
- [ ] Auth code referrer leak: callback page bare, `Referrer-Policy: no-referrer`
- [ ] CSRF binding: `state` param 綁 session cookie hash；consent screen POST only + anti-CSRF token
- [ ] Consent screen XSS: 禁 remote `logo_url`（先全禁 or proxy + cache）；strict CSP
- [ ] PKCE mandatory，no downgrade path
- [ ] Email enumeration constant-time on signup / login / forgot — login 404 time = Argon2 dummy compute
- [ ] Forgot password: reject if no local identity, return generic 200；若 Google-only user → email 發 "use Google sign-in" 私訊
- [ ] `password_resets` atomic: 新 request → invalidate prior unused rows for same user_id
- [ ] Session fixation: 不重用 pre-auth session cookie across auth boundary

**Infrastructure**
- [ ] Argon2id 參數: **benchmark Cloudflare Workers CPU 預算**（30s wall + 50ms default-tier burst），default OWASP desktop (m=64MB/t=3/p=4) 很可能 CPU throttle — 退 OWASP mobile (m=19456/t=2/p=1)
- [ ] Rate limit: `auth_rate_limits` 在 D1 是 footgun（write leader bottleneck）→ 改 Workers KV with TTL 或 Durable Object per key
- [ ] Email delivery: **Resend / Postmark / SES** — 不要 Cloudflare Email Routing（forward-only，outbound 是另一 beta service）
- [ ] Audit log → R2 / Logpush，D1 只保 30 天（防 row limit 爆）
- [ ] CORS per-client `allowed_origins` column（public SPA clients 需要）

**Backfill / Migration**
- [ ] `trip_permissions` distinct emails 建 `users` row：只 set `email_verified=1` for CF Access traceable emails；其他 `disabled_at=NOW` 等 claim-flow
- [ ] Migration 有 rollback sql + data preservation（`scripts/seed.sql` 8 個現有 emails）
