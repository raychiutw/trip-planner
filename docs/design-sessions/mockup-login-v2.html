登入頁規格摘要

  • 路由：/login — 兩狀態 view（未登入 form / 已登入 account）；不是 modal，是正式 page route
  • Sidebar nav 第 5 項：未登入 active 顯示 「登入」；登入後 sidebar 底部 chip 改顯示 account card（avatar + 名 + email），nav 第 5 項標籤可改為「帳號」
  • 未登入 desktop layout：50/50 split — 左 sign-in card（max 420px centered）+ 右 dark gradient 暖調 benefit copy（hero + 4 feature row + footnote）
  • OAuth providers：Google / Apple / LINE 三個（對齊 layout reference V2 OAuth provider plan + 適合台灣市場）
  • Email + password fallback：雙 field + 黑底 primary 「登入 →」button + 「忘記密碼 / 建立帳號」footer
  • 已登入 layout：左 profile card（avatar 80px + name + email + 3 stats）+ 右 settings 三 section（Profile / Preferences / Connected accounts）
  • Settings row 樣式：固定 label width 120px + value（muted）+ 右側 action button（編輯/變更/切換）或 toggle
  • Sign out：account header 右上 outline pill button（不放在 settings 列表，避免誤觸；視覺上 destructive 應更明顯但 V2 暫保守）
  • Mobile：sign-in 全寬 + bottom nav 常駐；form 順序由上至下：title hero → providers → divider → email/password → primary CTA → footer link → terms
  • 實作對應：V2 OAuth Server 任務（Workstream A · Phase 1-3）— 路由 + form UI 是 P1 起可做的部分；OAuth integration 在 P2-P3
  • Anti-slop check：未做 welcome cover 第一屏（DESIGN.md 禁止）— 第一屏即是 form，承載 primary action（登入），右側 benefit 是輔助 affordance 不是 title-only
