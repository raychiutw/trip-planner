Trip Modal v2 — OSM 欄位整合 mockup

對齊 V2 Terracotta SoT (docs/design-sessions/terracotta-preview-v2.html) · 補齊 9 個欄位讓 UI 建立的 trip 與 tp-create 結構齊平

Design Notes — 重要設計推斷（請 user 確認）
  • multi-dest vs region 欄位：現有 NewTripModal 已是 multi-dest list（沖繩 + 大阪 + 京都 拖拉排序），不是單一 input。本 mockup 解讀：region DB 欄位 = multi-dest names join 字串（例 "沖繩,大阪,京都"），不再單獨 input。OSM enrich 時依此切分查 Overpass。
  • title 自動命名規則：1 dest「2026 沖繩」/ 2 dests「2026 沖繩・大阪」/ 3+ dests「2026 沖繩等 3 地」。跨年用第一天年份。user 自填後不再覆蓋。
  • 進階設定 collapsible：新欄位（title / description / lang / travel_mode / self_drive）全收進「進階設定」folded，避免主畫面從 3 section 暴增到 7 section。預設收起，user 想動才展開。
  • published 改放底部 actions row：3-state segment「草稿 / 上線」緊貼建立按鈕，視覺上強調「user 必須選一個發布狀態」。預設草稿。
  • 保留現有 multi-dest + 雙日期模式：不破壞 user 已熟悉的 mindtrip 風 UX。
  • 拔除欄位：不再顯示 footer / food_prefs / og_description / auto_scroll / is_default 任何 input。
  • EditTripModal 共用 layout：跟 NewTripModal 99% 同，只差預填值 + region 變更時 title 旁顯示「更新提示」alert（user 主動點才覆寫，不自動）。
