探索頁規格摘要

  • 路由：/explore — search + 儲存池兩 tab；search 結果靠 POI search provider（Nominatim 免費 / Google Places 付費 — 待調查）
  • 2 個 tab：「搜尋」（active 動作）+「儲存池」（已 save 的 POI 列表）— 對齊 layout reference Q5 LOCKED
  • Search bar：rounded-full + icon + input + 區域選擇 chip（沖繩 · 那霸 ▾）
  • Category chips：全部 / 餐廳 / 景點 / 飯店 / 購物 / 寺廟 — 限 4-6 個（layout reference 建議「限 3-4 main category」，本 mockup 給 6 個含寺廟做 affordance demo）
  • POI card：4:3 cover + eyebrow（cat-dot + 類別 + 子類）+ title + rating + reviews + distance；右上 ♡ save button（hover/saved 變 accent fill）
  • 儲存池 mode 多選：左上換成 checkbox（☐ → ✓ accent fill），底部 sticky action bar「已選 N 個 · 加入到行程 ▾」
  • Sheet 展開（搜尋 mode）：cover image + eyebrow + POI title + meta chips + 位置/電話/時段/類型/筆記 + 「+ 加入行程 ▾」dropdown 列出所有 trips（atd-item: cover + title + meta）
  • 跟 /map 的分工：探索 = active search + save + add to trip；地圖 = passive 看你已有 trips 的地理 view（無 search、無 save、無 add）
  • Schema dependency：需新 saved_pois(id, user_id, poi_id, saved_at, note) table — FK 到既有 pois（POI Unification Phase 3 後 normalized）
  • Mobile 簡化：search bar 高度 42px、無 region chip、POI list 改橫 row（80×80 cover + body + 右上 save button）
  • 實作對應：B-P4 task scope（layout reference Q5）— 約 3 週（POI search API + 儲存池 UI + cross-link to trip promotion）
