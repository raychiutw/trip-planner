註冊頁規格摘要

  • 路由：/login/signup（Step 1 填表）/login/verify（Step 2 等 email 驗證）
  • Step 1：3 個 OAuth provider（Google/Apple/LINE 真實 logo SVG）+ Email 註冊三 field（顯示名稱 / Email / 密碼）+ 即時 password rule meter + terms checkbox（預設 checked）+ primary CTA「建立帳號」
  • Step 2 verify state：72px accent-subtle icon + 「檢查信箱」hero + 顯示已寄到的 email chip + 雙 action（「開啟 email app」primary / 「重寄連結」secondary）+ help footer（垃圾信件提示 / 改用其他 email）
  • Provider buttons：跟 login 同樣 3 個 OAuth 真實 logo SVG（Google 4 色、Apple 黑、LINE #06C755 綠）— 對齊品牌 guideline 要求
  • 顯示名稱欄位：mockup 多一個 field（login 沒有）— 旅伴會看到這個名字，後續可改
  • Password rules 即時驗證：3 條規則 dot + 文案（>=8 字 / 至少 1 數字 / 至少 1 大寫），達成的變 accent
  • Terms 行：accent-fill checkbox（預設勾選 — 但實作要 user 主動勾才符合 dark pattern 規避；mockup 為 demo 預勾）+ 雙 link 條款
  • 未驗證帳號 7 天清除：spec card 提到的 reassurance — 避免假帳號累積，符合 V2 OAuth Workstream A 的 hardening
  • Mobile：簡化 — 無右側 aside、無 password rule meter（僅 placeholder hint）、terms 縮短「同意條款與隱私權」
  • 實作對應：V2 OAuth Workstream A · Phase 2「Local password + email verification」
