Toast Redesign — Centering 修正 + Terracotta 對齊

原 toast container 已改 flex items-center modern centering，但 @keyframes toast-slide-down 仍含 translateX(-50%) legacy transform 把 bubble 往左推 50% → 截圖看到「該筆資料已經存在」被切到左半邊。 順帶把視覺對齊 DESIGN.md + terracotta-preview-v2.html 既有 .tp-status-toast / .tp-alert-panel pattern。

Root cause

css/tokens.css:840 的 @keyframes toast-slide-down 設 transform: translateX(-50%) translateY(-12px)，但 Toast.tsx:64 已改成 flex items-center 自動置中。 兩種 centering 機制疊加 → bubble 從 viewport 中心再往左偏 50% bubble 寬。修法：keyframes 拔掉 translateX(-50%) 只留 translateY。

decision brief

三個 variant 共用 container modern centering（position:fixed; top:safe-area; left:0; right:0; flex; items-center）+ keyframes 拔掉 translateX(-50%)。 差異只在 bubble 視覺權重。

V1 推薦 最忠於 mockup SoT；視覺最輕；4 type 共用 layout 只換 dot 色；長期維護成本最低。
V2 保守 跟既有 user 認知最一致；error 視覺權重最重；改動代碼最少（只動 keyframes + padding）。
V3 折衷 stripe + icon 雙訊號比 V1 強、比 V2 輕；跟 page-level alert-panel 風格 echo。
