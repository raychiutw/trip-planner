自訂景點 + 地址定位 — 3 個 mockup variants

Approach B：Map center pick + address typeahead 輔助。問題：旅伴加自訂 entry 時被 silent drop（無座標 → map 漏 + travel pill 空），這版本確保「保證有 coord 才能 submit」。3 個 variants 跨 form factor 比較。

Variant 1 · Mobile

Fullpage Route

/trip/:id/day/:n/add-custom-stop。IME 強制走全頁、keyboard 上時 map 暫退讓 dim。

開新分頁看細節 →

✓ 適合

  • 長 form / 多欄位
  • IME 高占用（注音輸入）
  • 需要 dedicated route 給 back button

✗ 代價

  • 新 route + MobileOnlyRoute guard
  • 離開 trip context（page swap）
  • back button 行為要對齊既有 AddStopPage
Variant 2 · Mobile

Bottom Sheet

Sheet 從底部上升、維持 trip 背景 dim 可見。鍵盤上 sheet shrink 自動避讓。

開新分頁看細節 →

✓ 適合

  • 保留 trip 視覺 context
  • 短任務感（drag-to-dismiss）
  • 對齊現代 mobile UI pattern

✗ 代價

  • Dynamic-height keyboard 處理棘手
  • Map 折疊狀態需特別設計 transition
  • 內容過多時 sheet 上推手感差
Variant 3 · Desktop

Two-pane Inline

AddStopPage「自訂」tab body 擴充，form 左 / map 右。無 IME 限制，map 可大。

開新分頁看細節 →

✓ 適合

  • ≥1024px 桌面、map 拖動空間大
  • 對齊既有 3-tab pattern
  • 無新 route / 無 reroute 邏輯

✗ 代價

  • 13" 多視窗會擠（min-width 1024 不算寬鬆）
  • Mobile 必須 reroute 到 V1 fullpage（兩條 path 維護）
